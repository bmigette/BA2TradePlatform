  <project_name>Deep Learning Financial Forecasting Platform</project_name>

  <overview>
    Build a comprehensive platform to train and evaluate deep learning models for financial forecasting.
    The platform consists of two main components:
    1. MODEL BUILDER - Uses genetic optimization to build best-fit models for financial timeseries prediction
    2. STRATEGY BACKTESTER - Tests trained models against historical market data to evaluate performance

    The application should provide a modern, fluid interface for configuring optimization jobs, monitoring
    training progress, visualizing datasets, and backtesting trading strategies based on model predictions.
  </overview>

      <class_imbalance_handling>
        Prediction targets often have severe class imbalance (e.g., 5% positive, 95% negative).
        A model predicting "negative" 100% of time gets 95% accuracy but is useless.

        User-configurable options in optimization job:

        <fitness_metric>
          The metric used by genetic algorithm to evaluate model quality:
          - accuracy: Standard accuracy (NOT recommended for imbalanced data)
          - f1_score: Harmonic mean of precision and recall (RECOMMENDED)
          - precision: Minimize false positives
          - recall: Minimize false negatives (catch all positive cases)
          - auc_roc: Area under ROC curve (good for ranking)
          - balanced_accuracy: Average of recall for each class
          Default: f1_score
        </fitness_metric>

        <loss_function>
          The loss function used during model training:
          - cross_entropy: Standard cross-entropy loss
          - weighted_cross_entropy: Auto-weighted based on class distribution
          - focal_loss: Down-weights easy examples, focuses on hard ones
            * gamma parameter (default 2.0): focusing strength
            * alpha parameter (auto): class weight balancing
          Default: focal_loss (best for imbalanced classification)
        </loss_function>

        <ui_warnings>
          - Show imbalance ratio in target statistics
          - Warning icon when positive class < 10%
          - Suggest focal_loss when imbalance detected
          - Display selected metric prominently in results (not accuracy)
        </ui_warnings>
      </class_imbalance_handling>

      <prediction_targets>
        - User-configurable output fields for each optimization job
        - Symmetric up/down predictions (must be balanced)
        - User specifies:
          * Profit percentage targets (e.g., 10%, 20%, 50%, etc.)
          * Maximum drawdown tolerance (e.g., 5%, 10%, 15%, etc.)
          * Time periods (e.g., 7, 15, 30, 90 days)
        - Example outputs:
          * price_up_10pct_5dd_7d (price goes up 10% with max 5% DD in 7 days)
          * price_down_10pct_5dd_7d (price goes down 10% with max 5% DD in 7 days)
        - User can define multiple output combinations per job
        - Constraint: Must be symmetric (if predict up scenarios, predict same down scenarios)
        - Model is optimized to predict the user-specified outputs, not to optimize the outputs themselves
      </prediction_targets>

      <trend_based_targets status="IMPLEMENTED">
        Alternative to price percentage targets - predict market trends/regimes.
        Provides more balanced class distribution than percentage-based targets.

        <trend_types>
          - Uptrend: Higher highs and higher lows over N periods
          - Downtrend: Lower highs and lower lows over N periods
          - Sideways/Consolidation: Price range bound within X% for N periods
          - Breakout Up/Down: Price moves beyond recent range by X%
        </trend_types>

        <trend_detection_methods>
          - moving_average: SMA crossovers (fast > slow = uptrend)
          - linear_regression: Slope of linear regression over N periods
          - adx: Average Directional Index for trend strength
          - pivot_points: Swing high/low analysis
          - donchian: Donchian channel breakout detection
        </trend_detection_methods>

        <configurable_parameters>
          - method: Detection method to use
          - lookback_period: Period for trend identification (default: 20)
          - prediction_horizon: How far ahead to predict (default: 5)
          - fast_period: Fast MA period for moving_average method (default: 10)
          - slow_period: Slow MA period for moving_average method (default: 30)
          - trend_strength_threshold: Minimum strength for ADX method (default: 25)
        </configurable_parameters>

        <output_columns>
          - trend_current: Current detected trend type
          - trend_target: Predicted trend N periods ahead
          - trend_strength: Numeric strength indicator
          - trend_is_uptrend: Binary (1/0) uptrend indicator
          - trend_is_downtrend: Binary (1/0) downtrend indicator
          - trend_is_sideways: Binary (1/0) sideways indicator
          - trend_class: Multi-class target (0=sideways, 1=uptrend, 2=downtrend)
